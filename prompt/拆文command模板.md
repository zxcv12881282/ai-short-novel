# 📚 短篇小说拆文分析Command

## 使用说明
直接对AI说：**"请用拆文command分析这篇小说"** 或 **"用拆文模板拆解这个故事"**

---

## 🎯 拆文分析框架（完整版）

### 第一部分：基础信息识别
```
1. 故事名称：[标题]
2. 全文字数：[约XXK]
3. 题材类型：[打脸爽文/复仇爽文/重生爽文/虐渣爽文等]
4. 核心冲突：[一句话概括主要矛盾]
5. 仿写/参考原文：[如有相似题材，指出]
```

### 第二部分：故事梗概
用200-300字概括完整故事：
- 主角是谁，什么身份
- 遇到什么问题/冲突
- 如何解决
- 最终结局

### 第三部分：故事结构（起承转合）
```
起：[开篇冲突，如何吸引读者]
  - 具体情节1
  - 具体情节2

承：[冲突升级，矛盾激化]
  - 具体情节1
  - 具体情节2

转：[反转时刻，真相揭露]
  - 具体情节1
  - 具体情节2

合：[收尾结局，善恶有报]
  - 具体情节1
  - 具体情节2
```

### 第四部分：人物分析
为每个主要角色建立档案：

**主角**
- 设定：[年龄、身份、背景]
- 性格：[3-5个关键词]
- 目标：[想达成什么]
- 行动：[主要行为路线]
- 成长弧光：[前后变化]

**反派/对手**
- 设定：[身份背景]
- 动机：[为什么这么做]
- 手段：[用了什么方法]
- 结局：[最终下场]

**配角**
- [关键配角的作用和意义]

### 第五部分：微创新点
这个故事与同类题材相比，有什么新意？
- 人物关系创新？
- 反转设计创新？
- 结局走向创新？
- 社会话题创新？

### 第六部分：悬念钩子
**开篇钩子**
- 具体是什么句子/情节
- 如何吸引读者继续看

**主线钩子**
- 钩子1：[什么悬念]
- 钩子2：[什么悬念]
- ...
- 如何环环相扣

### 第七部分：冲突升级（分层列出）
```
第1层：[初级冲突] - 情绪值30%
第2层：[冲突加剧] - 情绪值50%
第3层：[矛盾激化] - 情绪值70%
第4层：[濒临爆发] - 情绪值85%
第5层：[彻底爆发] - 情绪值100%
...

每一层说明：
- 具体发生了什么
- 为什么能激怒读者
- 如何为后面的爽点铺垫
```

### 第八部分：解决手段（核心爽点）
主角如何解决问题？用了什么方法？

分点列出：
1. **[方法名称]**：具体怎么做的，为什么爽
2. **[方法名称]**：具体怎么做的，为什么爽
3. ...

重点分析：
- 是否做到"以其人之道还治其人之身"
- 是否有智商碾压的爽感
- 是否有身份反转的惊喜

### 第九部分：写作技巧亮点
这篇文章用了哪些写作技巧？

可能的维度：
- **叙事视角**：第一人称/全知视角/限制视角
- **节奏把控**：快慢结合/情绪起伏
- **对话设计**：是否生动有张力
- **细节伏笔**：哪些细节埋了伏笔
- **特殊手法**：弹幕、直播、日记等
- **语言风格**：简洁/华丽/网感强

### 第十二部分：衍生选题（举一反三）
基于这个故事的套路，可以衍生哪些新选题？

给出3-5个选题：

**选题1：《标题》**
- 简要梗概（100字内）
- 核心冲突
- 反转点

**选题2：《标题》**
- 简要梗概（100字内）
- 核心冲突
- 反转点

...

### 第十三部分：总结提炼

**第一层：是什么**
用一段话总结整个故事的核心内容和情节走向

**第二层：为什么（爆款密码）**
分析为什么这个故事能成为爆款：

2. **[原因2]**：详细说明
3. **[原因3]**：详细说明
4. **[原因4]**：详细说明

**第三层：怎么用（创作启示）**

- 结构上的启示

## 💡 拆文注意事项

### 1. 完整性
- 确保读完整篇小说
- 不遗漏关键情节
- 理解作者真实意图

### 2. 客观性
- 既要看到优点
- 也要指出不足
- 不带个人偏见评价

### 3. 实用性
- 分析要可操作
- 选题要能落地
- 技巧要能学习

### 4. 结构化
- 分点清晰
- 逻辑连贯
- 便于查阅

### 5. 趣味性
- 使用emoji增加可读性
- 适当使用表格/列表
- 语言生动不枯燥
