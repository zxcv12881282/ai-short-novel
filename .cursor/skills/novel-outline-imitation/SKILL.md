# 小说大纲仿写工作流 (Novel Outline Imitation Workflow)

## 使用场景 (When to Use)

当用户想要**仿写一篇小说的结构**，并且提供了以下三个要素时：
1. 一篇原创参考小说（通常是一个 `.txt` 或 Markdown 文本）
2. 一段新的导语（故事的新背景和开篇）
3. 一个目标输出文件夹路径

## 核心工作流 (The Workflow)

这是一个严格的 **三步走** 序列任务。每次执行必须严格遵守顺序，等待前一步完成并生成对应的文件后，再将产出的内容作为参数传递给下一步。

### 第一步：大纲功能性拆解

1.  **准备**：读取本地提示词模板 `prompt/改稿prompt/大纲功能性拆解.md`。
2.  **替换**：将用户提供的【参考小说原文】替换到模板中“待分析文本”的代码块里（位于 `大纲功能性拆解.md` 末尾）。
3.  **执行与输出**：按照替换后模板的要求生成功能性拆解，并将结果保存到用户指定文件夹路径下的 `功能性拆解.md` 文件中。

### 第二步：大纲全方位拆解

1.  **准备**：读取本地提示词模板 `prompt/改稿prompt/大纲拆解prompt.md`。
2.  **替换**：将用户提供的【参考小说原文】替换到模板中“【在此下方粘贴完整短故事导语+正文】”的代码块里（位于 `大纲拆解prompt.md` 的末尾）。
3.  **执行与输出**：按照该模板（分为整体核心要素提取和逐章剧情拆解两阶段），生成全方位的大纲拆解。结果保存到用户指定文件夹路径下的 `全方位拆解.md` 文件中。

### 第三步：大纲仿写

1.  **准备**：读取本地提示词模板 `prompt/改稿prompt/大纲仿写.md`。
2.  **构建 Prompt**：
    *   将用户提供的【新导语】放入模板的 `新的导语：` 代码块中。
    *   将**第一步**生成的 `功能性拆解.md` 的全部内容放入模板的 `仿写的文章的大纲：` 代码块中。
    *   将**第二步**生成的 `全方位拆解.md` 的全部内容放入模板的 `补充信息：` 代码块中。
3.  **执行与输出**：严格按照 `大纲仿写.md` 中的“仿骨不仿皮”、情感超越和结构1:1对标的铁律执行仿写创作。将最终生成的全新大纲保存到用户指定文件夹路径下的 `仿写的大纲.md` 文件中。

## 注意事项 (Constraints & Rules)

*   **必须分步执行，不能遗漏产出文件**：用户明确要求将三个步骤骤的结果分别写入三个独立的 Markdown 文档。
*   **严禁修改提示词规则**：只能往固定的代码块内填入用户提供的小说或上一步产出的内容，不能更改原有提示词中的判断标准和系统限制（尤其是大纲仿写时的字数密度和不要扩写的要求）。
*   **严禁擅自加戏**：在第三步大纲仿写中，必须牢记 `大纲仿写.md` 的指令，基于新导语推演，绝不给角色擅自添加导语中未曾提及的设定（如职业）。

## Examples (对话示例)

**User Input:**
> 帮我执行小说大纲仿写。小说内容在 `@作品/范文.txt`，新导语是：“我是天界的仙女……直到新来的小师妹抢了我的风头”，文件路径放在 `@作品/仿写测试1/`

**Agent Actions:**
1.  读取 `范文.txt` 和 `大纲功能性拆解.md`。合成 Prompt 并生成内容，调用 Write 工具保存到 `@作品/仿写测试1/功能性拆解.md`。
2.  读取 `大纲拆解prompt.md`。合成 Prompt 并生成内容，调用 Write 工具保存到 `@作品/仿写测试1/全方位拆解.md`。
3.  读取 `大纲仿写.md`，使用刚生成的两份拆解文件和用户的导语合成最终的 Prompt，生成全新大纲，调用 Write 工具保存到 `@作品/仿写测试1/仿写的大纲.md`。
4.  回复用户任务已全部完成。
